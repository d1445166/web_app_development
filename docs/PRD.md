# 產品需求文件 (PRD) - 個人記帳簿

## 專案名稱
個人記帳簿 (Personal Expense Tracker)

## 專案描述
一款輕量化、專為個人設計的記帳系統，可以快速記錄每日消費與收入，幫助使用者追蹤財務狀況並提供餘額警示，以達到更良好的資金管理。

---

## 1. 專案概述

### 1.1 背景與動機
為了有效管理個人財務，需要一個簡單直觀的工具來記錄每天的收入與支出。市面上的記帳軟體通常功能繁雜，因此需要開發一款聚焦於核心需求、能清楚掌握餘額的個人化記帳系統，使理財過程輕鬆不費力。

### 1.2 目標用戶
- 自己（單一且個人的使用者）

### 1.3 核心價值主張
- **簡單快速**：極簡的操作流程，聚焦在當日收支與餘額。
- **直觀閱讀**：在單一頁面整合顯示當日完整收支數據。
- **財務掌握**：藉由餘額警示提醒，防止過度消費或透支。

---

## 2. 功能需求

以下列出系統的主要功能與對應的使用者故事 (User Stories)：

1. **記錄收入與支出**
   - *作為使用者，我希望可以輸入當天的收入與支出金額（包含項目說明），以便記錄資金的流動。*
2. **顯示餘額統計系統**
   - *作為使用者，我希望系統能由所有歷史紀錄自動計算並顯示目前的總餘額與統計，以便隨時了解自己的財務狀況。*
3. **以日期劃分資料**
   - *作為使用者，我希望記帳資料能依照日期進行分類與篩選，以便回顧過去或特定日期的收支明細。*
4. **當日收支顯示於同一頁面**
   - *作為使用者，我希望當天的所有收入與支出明細能直觀地整合顯示在同一個頁面上，以便一目了然今日的總結。*
5. **刪除錯誤的資料**
   - *作為使用者，我希望可以輕鬆刪除輸入錯誤或重複的收支紀錄，以便確保記帳資料正確無誤。*
6. **設定餘額過低警示**
   - *作為使用者，我希望可以設定一筆「最低餘額門檻」，當總餘額過低或低於此門檻時系統能發布警示，以便提醒自己節制消費。*

---

## 3. 非功能需求

### 3.1 技術限制
- **後端框架**：使用 Python 的 Flask 框架 (依據專案預設技術提案)
- **前端渲染**：使用 Jinja2 模板引擎與原生 HTML/CSS/JavaScript
- **資料庫**：使用輕量級的 SQLite 儲存使用者收支與設定資料

### 3.2 效能與安全考量
- **效能**：應用程式必須反應輕快，各項新增、刪除、查詢操作應於 1 秒內完成並更新畫面。
- **資料可靠性**：資料庫須置於安全的專案資料夾內並易於單機備份。
- **操作防呆**：在嘗試刪除收支紀錄時，應透過對話方塊（如 JavaScript `confirm()`）提供防呆確認機制以避免誤刪。

---

## 4. MVP 範圍 (Minimum Viable Product)

### Must Have (必須具備)
- 提供輸入介面，可以新增收入與支出（包含金額、類型、日期、備註說明）。
- 能在首頁單一畫面上顯示「當日」的收入、支出清單與「當下」的總金額餘額。
- 可以針對指定收支紀錄進行刪除動作，且刪除後能重新計算並顯示正確餘額。
- 支援以日期作為切換條件快速過濾與檢視歷史紀錄。

### Should Have (應該具備)
- 提供設定介面功能，可以自訂「餘額底線」。
- 在總餘額低於底線時，於頁面顯眼處跳出視覺化警示（例如：紅色警告橫幅或文字標示）。

### Nice to Have (未來期望)
- 支援收支出圖表統計分析（利用圓餅圖或長條圖顯示消費習慣）。
- 支援消費項目的細部分類（如：飲食、交通、娛樂等）。
- 資料匯出功能（匯出為 Excel 或是 CSV，方便二次處理）。
