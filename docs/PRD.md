# 產品需求文件 (PRD) - 作業提醒管理系統

## 專案名稱
作業提醒管理系統 (Assignment Reminder System)

## 專案描述
一款專為大學生設計的應用程式，旨在幫助學生集中管理多門課程的作業、報告及考試截止日，並透過多方通訊軟體進行即時提醒，降低忘記截止日期的風險，同時協助使用者提早規劃時間。

---

## 1. 專案概述

### 1.1 目標用戶
- **大學生**：需要同時管理多門課程作業、報告、考試時間的學生。

### 1.2 核心目標
- **目標一：提供作業、考試與報告的集中管理功能**
- **目標二：透過提醒通知，降低忘記截止日期的情況**
- **目標三：幫助使用者提早規劃時間，避免臨時抱佛腳**
- **目標四：簡單好操作的頁面，提升使用便利性**

---

## 2. 主要功能需求

### 2.1 截止日期追蹤與定時任務
- **功能描述**：系統能自動抓取（例如透過校園系統/LMS API）或由使用者手動設定作業、報告及考試的截止日期。
- **提醒設計**：允許使用者設定特定的時間點（如提前1天、提前3天、提前3小時等），系統會在指定時間點自動觸發提醒排程。

### 2.2 即時同步繳交狀態
- **功能描述**：提供狀態管理機制，讓使用者能清楚追蹤各項任務的完成度。
- **智慧停止提醒**：若學生已完成繳交（系統自動偵測或使用者手動勾選「已完成」），系統應立即停止針對該項作業的後續提醒通知，避免無效或過度的訊息打擾。

### 2.3 多方通訊軟體即時推送服務
- **功能描述**：整合外部 API（例如 LINE Notify、Gmail 等），提供多管道的訊息推送服務。
- **跨平台送達**：確保提醒訊息能打破應用程式本身的限制，直接送達學生的個人行動裝置或常用通訊軟體，確保重要截止日不錯過。

---

## 3. 非功能需求

### 3.1 介面與體驗設計 (UI/UX)
- **極簡操作**：頁面排版需乾淨、直覺，確保使用者能一眼看出即將到期、最緊急的任務。
- **響應式設計**：需支援手機版與電腦版操作，滿足大學生在不同場景下的使用需求。

### 3.2 系統與技術架構
- **即時性**：狀態更新（如勾選已完成）需即時生效，並立刻更新至資料庫與取消排程通知。
- **API 整合穩定性**：串接外部通訊軟體 API 時需處理好錯誤重試機制，並保留未來擴充其他平台（如 Telegram、Discord）的彈性。
